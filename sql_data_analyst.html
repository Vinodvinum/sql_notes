Analyst Workflow (Question To Query)

Use this sequence before writing SQL so your answer stays business-correct.

Workflow

  • Identify metric: count, sum, average, conversion, retention.
  • Identify grain: one row per customer, order, month, or product.
  • Choose source tables and join keys from schema.
  • Apply filters in WHERE, not in HAVING unless grouped.
  • Aggregate with GROUP BY, then validate totals.
  • Sort and limit for reporting output.

Template 1: KPI Summary

Syntax

SELECT
  DATE_TRUNC('month', order_date) AS month,
  COUNT(*) AS total_orders,
  COUNT(DISTINCT customer_id) AS active_customers,
  SUM(amount) AS revenue,
  AVG(amount) AS avg_order_value
FROM orders
WHERE order_date >= DATE '2024-01-01'
GROUP BY DATE_TRUNC('month', order_date)
ORDER BY month;

Use Case

Monthly business performance report.

Template 2: Top Customers

Syntax

SELECT
  c.customer_id,
  c.company_name,
  SUM(o.amount) AS total_spend,
  COUNT(o.order_id) AS total_orders
FROM customers c
JOIN orders o ON o.customer_id = c.customer_id
WHERE o.order_date BETWEEN DATE '2024-01-01' AND DATE '2024-12-31'
GROUP BY c.customer_id, c.company_name
ORDER BY total_spend DESC
LIMIT 10;

Explain Like Real

I join customer and order data, filter to target year, aggregate spend, then rank by spend to return top 10.

Template 3: Category Contribution %

Syntax

WITH category_sales AS (
  SELECT
    p.category_id,
    SUM(od.quantity * od.unit_price) AS sales
  FROM order_details od
  JOIN products p ON p.product_id = od.product_id
  GROUP BY p.category_id
)
SELECT
  category_id,
  sales,
  ROUND(100.0 * sales / SUM(sales) OVER (), 2) AS contribution_pct
FROM category_sales
ORDER BY contribution_pct DESC;

Use Case

Find which categories drive most revenue.

Template 4: Cohort Retention (Basic)

Syntax

WITH first_month AS (
  SELECT
    customer_id,
    DATE_TRUNC('month', MIN(order_date)) AS cohort_month
  FROM orders
  GROUP BY customer_id
),
activity AS (
  SELECT
    o.customer_id,
    DATE_TRUNC('month', o.order_date) AS activity_month
  FROM orders o
)
SELECT
  f.cohort_month,
  a.activity_month,
  COUNT(DISTINCT a.customer_id) AS retained_customers
FROM first_month f
JOIN activity a ON a.customer_id = f.customer_id
GROUP BY f.cohort_month, a.activity_month
ORDER BY f.cohort_month, a.activity_month;

Use Case

Track customer retention by cohort month.

Template 5: Running Total and Previous Value

Syntax

SELECT
  order_date,
  amount,
  SUM(amount) OVER (ORDER BY order_date) AS running_revenue,
  LAG(amount) OVER (ORDER BY order_date) AS prev_amount,
  amount - LAG(amount) OVER (ORDER BY order_date) AS delta_vs_prev
FROM daily_sales
ORDER BY order_date;

Use Case

Dashboard trend analysis and day-over-day change.

Template 6: Data Quality Check

Syntax

SELECT
  COUNT(*) AS total_rows,
  COUNT(*) - COUNT(customer_id) AS null_customer_id,
  COUNT(*) - COUNT(order_date) AS null_order_date,
  SUM(CASE WHEN amount <= 0 THEN 1 ELSE 0 END) AS invalid_amount_rows
FROM orders;

Use Case

Quick health check before analysis or dashboard build.
