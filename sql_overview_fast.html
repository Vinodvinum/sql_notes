SQL Overview Fast - Detailed Explanation

This page explains SQL syntax, how to use each concept, and where to use it in interviews and real analysis tasks. It is designed for you to explain your logic while solving.

Source: practice.txt Mode: detailed Focus: syntax + usage + interview explanation

SELECT and FROM

Syntax

SELECT column1, column2
FROM table_name;

How To Use

Choose only needed columns to keep output clean and improve performance.

Where To Use

Any query that reads data from one or more tables.

WHERE Filtering

Syntax

SELECT *
FROM table_name
WHERE condition;

How To Use

Apply row-level conditions before grouping or sorting.

Where To Use

When question asks find, show, list, or only rows matching criteria.

ORDER BY and LIMIT

Syntax

SELECT *
FROM table_name
ORDER BY column_name DESC
LIMIT 5;

How To Use

Sort first, then keep top or bottom rows with LIMIT.

Where To Use

Top N, latest records, highest or lowest value questions.

JOIN

Syntax

SELECT a.col, b.col
FROM table_a a
JOIN table_b b ON a.key = b.key;

How To Use

Use aliases and always define join condition ON key columns.

Where To Use

When the answer needs columns from multiple related tables.

GROUP BY and Aggregates

Syntax

SELECT city, COUNT(*) AS total
FROM patients
GROUP BY city;

How To Use

Group rows by a category and apply COUNT, SUM, AVG, MIN, or MAX.

Where To Use

Summary questions such as total by city, average by category.

HAVING

Syntax

SELECT city, COUNT(*) AS total
FROM patients
GROUP BY city
HAVING COUNT(*) > 5;

How To Use

Use HAVING to filter grouped results. WHERE cannot filter aggregate output.

Where To Use

Only-once, duplicates, more-than-N group conditions.

CASE

Syntax

SELECT name,
CASE
  WHEN score >= 80 THEN 'High'
  WHEN score >= 50 THEN 'Medium'
  ELSE 'Low'
END AS level
FROM marks;

How To Use

Map business rules into readable labels in query output.

Where To Use

Status classification, bands, flags, or conditional metrics.

UNION

Syntax

SELECT col1, col2 FROM table_a
UNION
SELECT col1, col2 FROM table_b;

How To Use

Combine same-shape result sets. Use UNION ALL if duplicates are allowed.

Where To Use

Merge records from similar tables like patients and doctors.

DISTINCT

Syntax

SELECT DISTINCT city
FROM patients;

How To Use

Remove duplicate values from selected columns.

Where To Use

Unique categories, years, cities, names.

EXISTS and NOT EXISTS

Syntax

SELECT *
FROM patients p
WHERE NOT EXISTS (
  SELECT 1 FROM admissions a
  WHERE a.patient_id = p.patient_id
);

How To Use

Check whether related rows exist or do not exist in another table.

Where To Use

Unmatched records, anti-join logic, active or inactive entities.

LIKE, IN, BETWEEN

Syntax

WHERE name LIKE 'C%'
WHERE city IN ('London', 'Paris')
WHERE weight BETWEEN 100 AND 120

How To Use

LIKE for patterns, IN for multiple exact values, BETWEEN for ranges.

Where To Use

Text filtering, list filtering, numeric or date ranges.

NULL Handling

Syntax

WHERE allergies IS NULL
WHERE allergies IS NOT NULL
UPDATE patients SET allergies = 'NKA'
WHERE allergies IS NULL;

How To Use

Use IS NULL and IS NOT NULL, never equals NULL.

Where To Use

Missing values, cleanup tasks, quality checks.

Date and String Functions

Syntax

strftime('%Y', birth_date)
UPPER(last_name)
LOWER(first_name)
LENGTH(first_name)

How To Use

Extract year or format text directly in query output.

Where To Use

Time-based reporting and output formatting questions.

Interview Explanation Cheat Sheet

Step 1 Explain requirement first: table source, filter logic, expected output.
Step 2 Explain query blocks in order: FROM/JOIN, WHERE, GROUP BY, HAVING, ORDER.
Step 3 Mention validation: row count check, null check, sort check.
Common Mistake Mixing WHERE and HAVING, or missing JOIN keys.
Quick Rule WHERE filters rows before grouping; HAVING filters groups after aggregation.
Practice Pattern Read question, choose pattern, write syntax skeleton, then fill columns.
