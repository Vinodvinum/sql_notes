Phase 1: Query Basics

Foundation
  • Topics: SELECT, INSERT, UPDATE, DELETE, WHERE
  • Goal: read, modify, and filter rows safely
  • Common mistake: UPDATE or DELETE without WHERE

Phase 2: Query Filtering

Most Important
  • Topics: JOIN, GROUP BY, HAVING, CASE, DISTINCT, EXISTS
  • Goal: combine tables and summarize business data
  • Interview value: solves most real analyst questions

Phase 3: Functions

Advanced Analysis
  • Aggregate: COUNT, SUM, AVG, MAX, MIN
  • Window: LAG, LEAD, FIRST_VALUE, LAST_VALUE
  • String, numeric, and date functions for dashboard logic

Phase 4: Tables & Constraints

Schema Control
  • Topics: CREATE, ALTER, DROP, data types
  • Constraints: PK, FK, UNIQUE, CHECK, DEFAULT, INDEX
  • Goal: design reliable relational structures

Fast Memory Map

WHEREFilter raw rows before grouping.
JOINConnect related tables through keys.
GROUP BYAggregate rows into summary groups.
HAVINGFilter aggregated groups.
CASEApply business rules in query output.
UNIONMerge result sets with compatible columns.

  1. Days 1-2: Query basics and safe filtering.
