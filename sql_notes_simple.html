Chapter 1

What is Data, Database, and Why It Exists

Step 1: Very simple meaning

Data means any information.

  • Name -> Vinod
  • Age -> 24
  • Salary -> 30000
  • City -> Mysore

All these are data.

Step 2: Amazon story

Amazon has customers, orders, products, payments. Each of these creates data every second.

If that data is scattered in random files, finding anything becomes slow and confusing.

Step 3: What is a database?

A database is an organized system used to store data so we can access and manage it easily.

Simple analogy: database is like a library, tables are like books, data is like the content inside those books.

Interview answer

A database is an organized collection of data used to store, retrieve, and manage information efficiently.

Chapter 2

Tables, Rows, Columns

Step 1: What is a table?

A table is where data is stored inside a database. Think of it like an Excel sheet.

Step 2: Columns and rows

Columns are types of information. Rows are actual records.

  • Column: customer_id, name, city
  • Row: one full customer record

Simple memory trick

Column = what type of data?

Row = actual data.

Interview answer

A table stores data in rows and columns, where each row is a record and each column is an attribute.

Simple meaning: SQL is the language used to communicate with a database.

Interview answer

SQL is used to communicate with databases. It helps us retrieve, filter, analyze, and manage data efficiently.

Chapter 4

SELECT: Your First SQL Command

Business situation

Your manager says: "Show me customer data."

SELECT * FROM customers;

Break it down

SELECT -> what you want

* -> everything

FROM -> from which table

Meaning: give me all data from customers table.

SELECT name FROM customers;
SELECT name, city FROM customers;

Interview answer

SELECT is used to retrieve data from a database. We can fetch all columns or only specific columns.

Mini practice

Q1: Write SQL to show only the city column.

Chapter 5

WHERE: Filtering Data

Business situation

Your manager says: "Show only customers from Bangalore."

SELECT * FROM customers
WHERE city = 'Bangalore';

Break it down

WHERE adds a condition.

Meaning: give only those rows where city is Bangalore.

WHERE city = 'Mysore'
WHERE salary > 40000
WHERE age < 30
WHERE city != 'Delhi'

Interview answer

WHERE is used to filter records based on conditions so that only relevant data is returned.

Mini practice

Q1: Show employees with salary greater than 40000.

Chapter 6

AND, OR, NOT: Combining Conditions

Business situation

Your manager says: "Show customers from Bangalore and salary greater than 40000."

SELECT * FROM customers
WHERE city = 'Bangalore'
  AND salary > 40000;
SELECT * FROM customers
WHERE city = 'Mysore'
   OR city = 'Bangalore';
SELECT * FROM customers
WHERE NOT city = 'Delhi';

Simple meaning

AND means both conditions must be true.

OR means at least one condition must be true.

NOT means exclude or reverse condition.

Interview answer

AND returns rows where all conditions are true, while OR returns rows where at least one condition is true.

Mini practice

Q1: Show employees from Mysore and salary greater than 30000.

Chapter 7

ORDER BY and LIMIT

Business situation

Your manager says: "Show me highest paying customers."

SELECT * FROM customers
ORDER BY salary DESC;

Simple meaning

ORDER BY means sorting.

DESC means highest to lowest.

ASC means lowest to highest.

SELECT * FROM customers
ORDER BY salary DESC
LIMIT 3;

Interview answer

ORDER BY is used to sort the result set, and LIMIT is used to restrict how many rows we want to see.

Q1: Show employees sorted by age from low to high.

Chapter 11

Aggregation Functions

Simple meaning

Aggregation means combining many rows into one result.

SELECT COUNT(*) FROM customers;
SELECT SUM(salary) FROM employees;
SELECT AVG(salary) FROM employees;
SELECT MAX(salary) FROM employees;
SELECT MIN(salary) FROM employees;

Interview answer

Aggregate functions perform calculations on multiple rows and return one value, such as COUNT, SUM, AVG, MAX, and MIN.

Mini practice

Count the number of employees.

Chapter 12

GROUP BY

Business situation

Your manager says: "Show me revenue per city."

SELECT city, COUNT(*) AS total_customers
FROM customers
GROUP BY city;
SELECT city, SUM(amount) AS revenue
FROM orders
GROUP BY city;

Interview answer

GROUP BY groups rows based on a column so that aggregate functions can be applied to each group.

Mini practice

Count employees per city.

Chapter 13

HAVING

Simple meaning

HAVING filters grouped results.

WHERE filters rows before grouping.

SELECT city, COUNT(*) AS total_employees
FROM employees
GROUP BY city
HAVING COUNT(*) > 2;

Interview answer

WHERE filters rows before grouping, while HAVING filters aggregated results after GROUP BY.

Mini practice

Show cities where employee count is more than 2.

INNER JOIN

SELECT c.name, o.amount
FROM customers c
INNER JOIN orders o
ON c.customer_id = o.customer_id;

Simple meaning

INNER JOIN returns only matching rows from both tables.

Interview answer

INNER JOIN returns only the records that have matching values in both tables.

Mini practice

Chapter 16

LEFT JOIN

SELECT c.name, o.amount
FROM customers c
LEFT JOIN orders o
ON c.customer_id = o.customer_id;

Simple meaning

LEFT JOIN keeps all rows from the left table. If no match is found, right-side columns become NULL.

Interview answer

LEFT JOIN returns all records from the left table and matching records from the right table. If there is no match, NULL is returned.

Chapter 17

Subqueries

Simple meaning

A subquery is a query inside another query.

SELECT *
FROM employees
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
);

Interview answer

A subquery is a nested query used to perform intermediate filtering or calculation.

Mini practice

Find employees with salary less than average salary.

Chapter 18

CASE WHEN

Simple meaning

CASE WHEN works like if-else inside SQL.

SELECT name, salary,
CASE
    WHEN salary >= 40000 THEN 'High'
    WHEN salary >= 20000 THEN 'Medium'
    ELSE 'Low'
END AS salary_type
FROM employees;

Interview answer

CASE WHEN is used to apply conditional logic in SQL and return different values based on conditions.

Mini practice

Classify salary into High, Medium, Low.

Chapter 19

Window Functions

Business situation

Your manager says: "Rank employees by salary" or "Show running total of sales."

Simple meaning

Window functions perform calculations across rows without collapsing the result like GROUP BY does.

Basic syntax:

FUNCTION() OVER (PARTITION BY column ORDER BY column)
SELECT name, salary,
RANK() OVER (ORDER BY salary DESC) AS rnk
FROM employees;
SELECT order_id, amount,
SUM(amount) OVER (ORDER BY order_id) AS running_total
FROM orders;

Interview answer

Window functions perform calculations across related rows while still keeping each original row in the result.

Mini practice

Rank employees by salary.

Add-on

RANK vs DENSE_RANK vs ROW_NUMBER

Function What happens in ties? Simple meaning
RANK() Same rank for ties, next rank is skipped 1, 2, 2, 4
DENSE_RANK() Same rank for ties, no skipping 1, 2, 2, 3
ROW_NUMBER() No same rank, always unique sequence 1, 2, 3, 4

Strong interview answer

RANK gives the same rank to duplicate values and skips the next rank, DENSE_RANK does not skip ranks, and ROW_NUMBER always gives unique sequential numbers.

Phase 6

Real business problems end to end

This is where everything comes together. Now you think like an analyst, not just like someone who knows syntax.

Problem 1

Top customers by revenue

Business question

Find top 3 customers based on total order amount.

Thinking

Revenue per customer -> SUM

Group -> customer

Sort -> DESC

Top 3 -> LIMIT

SELECT c.name, SUM(o.amount) AS total_revenue
FROM customers c
INNER JOIN orders o
ON c.customer_id = o.customer_id
GROUP BY c.name
ORDER BY total_revenue DESC
LIMIT 3;

Interview explanation

First, join customers and orders using customer_id. Then aggregate total revenue per customer using SUM. Finally, sort descending and limit to top 3.

Problem 2

Customers with no orders

Business question

Find customers who never placed an order.

Thinking

Need all customers -> LEFT JOIN

Missing orders -> NULL

SELECT c.name
FROM customers c
LEFT JOIN orders o
ON c.customer_id = o.customer_id
WHERE o.customer_id IS NULL;
Problem 3

Average order value

Business question

What is the average order amount?

SELECT AVG(amount) FROM orders;

Explanation

AVG calculates the average value of all orders.

Problem 4

Revenue per city

Business question

Which city generates the highest revenue?

Thinking

Need customer table plus orders table.

Group by city.

SUM revenue.

SELECT c.city, SUM(o.amount) AS revenue
FROM customers c
INNER JOIN orders o
ON c.customer_id = o.customer_id
GROUP BY c.city
ORDER BY revenue DESC;
Problem 5

Second highest salary

Thinking

Rank salaries.

Pick rank = 2.

SELECT name, salary
FROM (
    SELECT name, salary,
           RANK() OVER (ORDER BY salary DESC) AS rnk
    FROM employees
) t
WHERE rnk = 2;

Explanation

Use a window function to rank employees by salary, then filter rank 2 to get the second highest salary.

Important Extras

Primary key, foreign key, constraints, and data types

These are core interview topics. They are simple once you see the role clearly.

Extra 1

Primary Key

Concept

Primary key uniquely identifies each row.

No duplicates. No NULL.

CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    name VARCHAR(50),
    city VARCHAR(50)
);

Interview answer

Primary key uniquely identifies each record and cannot contain NULL or duplicate values.

Extra 2

Foreign Key

Concept

Foreign key links one table to another table.

CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    amount INT,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

Interview answer

Foreign key maintains relationship between tables and supports referential integrity.

Interview Bank

Question, answer, and exactly how to say it in interview

This section is for speaking practice. Read these answers out loud. The goal is to sound clear, simple, and confident.

How To Speak

Simple interview speaking formula

  1. Start with one clear definition.
  2. Then say why it is used.
  3. If needed, give one small example.
  4. Stop there. Do not make the answer too long.

Best speaking style

Speak like this: short sentence, clear meaning, one useful point, one example if needed.

Interview Q1

What is SQL?

Short answer

SQL is a language used to communicate with relational databases.

Exactly how to say it

"SQL stands for Structured Query Language. It is used to communicate with relational databases. With SQL, we can retrieve, filter, analyze, insert, update, and delete data."

Interview Q2

What is a database?

Short answer

A database is an organized collection of data.

Exactly how to say it

"A database is an organized collection of data that helps us store, retrieve, and manage information efficiently. It is used when data becomes large and needs to be handled in a structured way."

Interview Q3

What is a table?

Short answer

A table stores data in rows and columns.

Exactly how to say it

"A table is a structured format inside a database where data is stored in rows and columns. Each row represents one record, and each column represents one attribute of that record."

Interview Q4

What does SELECT do?

Short answer

SELECT is used to retrieve data from a table.

Exactly how to say it

"SELECT is used to retrieve data from a database table. We can use it to fetch all columns or only specific columns depending on the business requirement."

Interview Q5

What is the WHERE clause?

Short answer

WHERE filters rows based on a condition.

Exactly how to say it

"WHERE clause is used to filter records based on specific conditions. It helps us retrieve only relevant rows instead of the full table."

Interview Q6

Difference between AND and OR?

Short answer

AND needs all conditions true. OR needs at least one condition true.

Exactly how to say it

"AND returns rows only when all given conditions are true, whereas OR returns rows when at least one condition is true."

Interview Q7

What is ORDER BY?

Short answer

ORDER BY is used to sort the result.

Exactly how to say it

"ORDER BY is used to sort the result set based on one or more columns. We can sort in ascending order using ASC or descending order using DESC."

Interview Q8

What are aggregate functions?

Short answer

They calculate one result from many rows.

Exactly how to say it

"Aggregate functions perform calculations on multiple rows and return a single value. Common examples are COUNT, SUM, AVG, MAX, and MIN."

Interview Q9

What is GROUP BY?

Short answer

GROUP BY groups rows so aggregate functions can be applied per group.

Exactly how to say it

"GROUP BY is used to group rows based on a column, so that aggregate functions can be applied to each group separately. For example, we can count employees city-wise or calculate revenue branch-wise."

Interview Q10

Difference between WHERE and HAVING?

Short answer

WHERE filters rows. HAVING filters grouped results.

Exactly how to say it

"WHERE is used to filter rows before grouping, while HAVING is used to filter aggregated results after GROUP BY."

Interview Q11

Why do we use JOIN?

Short answer

JOIN combines data from multiple tables.

Exactly how to say it

"JOIN is used to combine data from multiple tables based on a common column. It is useful when related information is stored separately, like customer data in one table and order data in another."

Interview Q12

What is INNER JOIN?

Short answer

INNER JOIN returns only matching rows from both tables.

Exactly how to say it

"INNER JOIN returns only the records that have matching values in both tables based on a common column."

Interview Q13

What is LEFT JOIN?

Short answer

LEFT JOIN keeps all rows from the left table and matched rows from the right table.

Exactly how to say it

"LEFT JOIN returns all records from the left table and matching records from the right table. If no match is found, NULL values are returned for the right side."

Interview Q14

What is a subquery?

Short answer

A subquery is a query inside another query.

Exactly how to say it

"A subquery is a query nested inside another SQL query. It is used for intermediate calculations, filtering, or comparisons."

Interview Q15

What is CASE WHEN?

Short answer

CASE WHEN applies conditional logic in SQL.

Exactly how to say it

"CASE WHEN is used to apply conditional logic in SQL and return different values based on conditions, similar to if-else statements in programming."

Interview Q16

What is a primary key?

Short answer

Primary key uniquely identifies each row.

Exactly how to say it

"Primary key is a column or set of columns that uniquely identifies each record in a table. It cannot contain NULL values or duplicate values."

Interview Q17

What is a foreign key?

Short answer

Foreign key links one table to another table.

Exactly how to say it

"Foreign key is a column used to create a relationship between two tables. It usually refers to the primary key of another table and helps maintain referential integrity."

Interview Q18

Difference between DELETE and TRUNCATE?

Short answer

DELETE removes selected rows. TRUNCATE removes all rows from the table.

Exactly how to say it

"DELETE is used to remove specific rows, usually with a WHERE condition, whereas TRUNCATE is used to remove all rows from a table quickly."

Interview Q19

What is a window function?

Short answer

Window functions calculate across related rows without collapsing the result.

Exactly how to say it

"Window functions perform calculations across a set of rows related to the current row without reducing the result set. They are useful for ranking, running totals, and row-wise comparisons."

Interview Q20

How comfortable are you with SQL?

Exactly how to say it

"I am comfortable with core SQL concepts like SELECT, filtering with WHERE, sorting, joins, aggregate functions, GROUP BY, and HAVING. I can also work with subqueries, CASE WHEN, and basic window functions like RANK and running totals. I am especially comfortable solving analysis-style questions such as top customers, revenue by group, and missing data scenarios."
